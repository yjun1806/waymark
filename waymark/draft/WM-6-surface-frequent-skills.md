# WM-6 · 자주 쓰는 설치 스킬을 예시로 보여주기

## Why

"다른 도구를 쓰세요"([[WM-5]])나 init 마무리 안내는 **추상적**이면 잘 안 와닿는다. 사용자
환경에 실제로 **설치된 스킬 중 자주 쓰는 것 몇 개를 구체 예시로** 보여주면, "내용은 이걸로
채우면 되겠구나"가 즉시 잡힌다. waymark이 "형식만 다루고 도구는 네 것 그대로"라는 메시지를
말이 아니라 **눈앞의 예시**로 전한다. 인프라/툴링 — planning source 없음.

## How

설치된 스킬을 감지해 몇 개를 예시로 제시한다. 쓰이는 곳:
- [[WM-5]]의 "(b) 다른 도구로 채우기" 프롬프트의 보기 목록
- `waymark-init` Step 7 핸드오프의 "내용은 이런 도구로 채우면 됩니다" 안내

감지 소스(택1/조합):
- `~/.claude/plugins/`의 설치·활성 스킬 목록(플러그인 캐시/설정)
- 세션에 노출된 available skills

**"자주 쓰는"의 판정**이 핵심 난점 — 사용량 카운트를 직접 알 수 없다. 후보:
- (A) `.waymark.yml`에 선호 도구 shortlist를 선언(`tools: [...]`) — 명시적, 예측가능
- (B) 설치된 스킬 이름을 그냥 몇 개 나열(무판정) — 단순하지만 관련성 낮을 수 있음
- (C) 휴리스틱(최근 세션 등) — 복잡, 접근 제약

## Tasks

- [ ] T1 설치 스킬 감지 로직(소스 결정)
- [ ] T2 "자주 쓰는" 판정 방식 확정(A/B/C)
- [ ] T3 WM-5 프롬프트·init Step 7에 예시 주입
- [ ] T4 예시 0개(감지 실패/미설치) 시 일반 안내로 폴백

## Decisions

- D1 이 기능은 **표시(surfacing)만** — 스킬을 대신 실행하지 않는다. 예시 제시가 전부.
- Open Q "자주 쓰는" 판정 = A(config shortlist) 권장(단순·예측가능). B는 폴백, C는 보류.
- Open Q 예시 개수 상한(3~5개?)과, 무관한 waymark 자체 스킬(init/remove/update) 제외 규칙.
