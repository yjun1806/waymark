# WM-5 · work 문서 생성 시 내용 채우기를 다른 도구에 위임할지 묻기

## Why

waymark의 가치는 **형식/컨벤션**(이슈 하나=문서 하나, 폴더=상태, 참조-비복제)이지 **내용
authoring 도구**가 아니다. "여러분이 쓰던 도구는 그대로 쓰세요 — 이 방식은 결과물의 형식만
다룬다"가 핵심 철학이다.

그렇다면 work 문서를 만들 때, `Why`/`How`/`Tasks` **내용**을 꼭 waymark(=지금 대화)로 채울 이유가
없다. Spec Kit이든, 다른 스킬/에이전트든, 사용자가 선호하는 흐름이든 — 내용은 그걸로 채우고
waymark은 **껍데기(형식)만** 제공하면 된다. 그래서 waymark-work가 **"내용을 다른 도구로
채울래요?"를 물어보게** 한다. 인프라/툴링 — planning source 없음.

## How

`skills/waymark-work/SKILL.md`에서 템플릿으로 doc을 스캐폴딩한 **직후** 프롬프트 추가:

> 내용(Why/How/Tasks)을 어떻게 채울까요?
> - (a) 지금 이 대화로 (기본, 현행)
> - (b) 다른 도구/스킬로 — waymark은 **형식만**(frontmatter·4헤딩·폴더 배치) 세팅하고, 내용
>   authoring은 그 도구에 맡김

(b)를 고르면 waymark은 스캐폴드(빈 4헤딩 + frontmatter)만 남기고 "이제 `<도구>`로 채우세요"로
**핸드오프**한다 — 내용 생성을 **재구현하지 않는다**(format-only 유지). 어떤 도구를 예시로
보여줄지는 [[WM-6]](자주 쓰는 설치 스킬 예시)가 공급.

**불변식:** 형식(헤딩·frontmatter·폴더=상태)은 어느 경로든 waymark 소유. 위임되는 건 **내용**뿐.

## Tasks

- [ ] T1 waymark-work 스캐폴딩 직후 (a)/(b) 프롬프트 추가
- [ ] T2 (b) 경로 — 형식만 남기고 도구로 핸드오프하는 안내 문구
- [ ] T3 예시 도구 목록은 [[WM-6]] 연동(있으면 사용, 없으면 일반 안내)
- [ ] T4 README/ARCHITECTURE에 "형식만 강제, 내용은 BYO 도구" 한 줄 반영

## Decisions

- D1 waymark은 **format-only** — 내용 생성 로직을 갖지 않는다. 위임은 핸드오프(안내)이지 도구
  체이닝 구현이 아님(스킬 간 자동 호출은 범위 밖, 우선 안내로).
- D2 기본값 = (a) 지금 대화 — 현행 동작 보존, (b)는 명시 선택.
- Open Q 핸드오프를 "안내만" 할지, 나중에 실제 도구 실행 연계까지 갈지 — 우선 안내.
